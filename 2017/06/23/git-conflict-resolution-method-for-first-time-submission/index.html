Git 首次提交时冲突解决方法

发表于 2017-06-23 | 更新于 2018-10-05 | 分类于 Git |

问题描述

在远端创建一个项目时，选中使用README初始化项目。
两外在本地创建一个项目，创建文件并提交修改。
此时将本地的代码提交到远程仓库，执行以下代码后提示错误

结果提示：

git refusing to merge unrelated histories

解决方法

问题原因是本地的 master 分支上有修改，远程仓库的 master 分支有修改，但它们没有共同的父节点。所以 git 不知道如何处理。

解决方法是：

更新
git pull origin master -u --allow-unrelated-histories

之前版本

  1. 通过 git fetch origin master 将远程代码保存到本地的 origin/master 分支

  2. 使用 git rebase origin/master 将远程代码和本地代码进行合并，代码会提示以下信息

    First, rewinding head to replay your work on top of it…
    Applying: init
    Using index info to reconstruct a base tree…
    Falling back to patching base and 3-way merge…

可以看到内部操作是，回头重新执行版本的初始化工作，先进行初始化，使用索引信息重构 base tree，回到修补基地进行3路合并。

意思是 我重新创建一个空项目，然后将远程和本地的两个版本合并到主线上。

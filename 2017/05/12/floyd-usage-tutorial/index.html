第二部分 概念及命令介绍

floyd 管理

恭喜，你已经知道如何在 floyd 上执行代码了。接下来介绍下 floyd 的管理逻辑。先看看以下几个概念：

  • project （项目）

    我们执行 floyd init hello 就是初始化了 hello 项目。

  • experiments （实验）

    实验是在项目基础上进行的，为了让项目输出结果最佳，我们需要做很多次实验，所以一个项目可能有 N 个关联的实验。

  • module （模块）

    这个是 floyd 提供的公共模块，具体如何使用，还不清楚。

  • Data （数据）

    数据有两种，可能是我们上传，也可能是实验时生成的。

接下来分别介绍它们如何管理

项目管理

描述 命令
项目初始化 floyd init project_name
查看项目实验 floyd status

很奇怪没有删除项目的命令，大部分命令都是基于实验的。

实验管理

描述 命令
运行实验 floyd run --mode jupyter
删除实验 floyd delete runid
停止实验 floyd stop runid
实验信息 floyd info runid
实验日志 floyd logs runid

数据管理

描述 命令
项目数据 floyd data status
删除实验数据 floyd data delete data_id
初始化数据文件夹 floyd data init data_name
上传数据 floyd data upload
删除数据 floyd data delete data_id

腾讯IM登录和业务系统登录状态不一致问题及解决方法

Posted on 2018-10-02 | Edited on 2018-10-04 |

背景

项目中需要使用聊天功能，综合评估后选用腾讯IM。为了保证业务的正确性，需要保证用户任意时间点在腾讯IM和业务系统的登录状态一致。因此需要考虑多点登录相互踢的情况。当然，腾讯提供了多设备同时登录的解决方案，由于自身业务场景没有这方面的需要，多设备同时登录不在本文讨论范围。
另外，项目使用了“最小代价”的集成方式，直接将腾讯IM demo移植到项目中，这并不影响问题的讨论。

问题

我们遇到的问题是，同一用户在设备A上登录后，退出软件，然后在设备B上登录。此时打开设备A上软件，业务系统会检测到用户在其他设备登录，清空本地登录信息。用户输入账户信息重新登录，业务系统可以登录成功，腾讯IM登录失败。在设备B上可以正常使用聊天功能，设备A上可以进行与账户相关的操作，但是无法收到最新的聊天信息。

Git 首次提交时冲突解决方法

Posted on 2017-06-23 | Edited on 2018-10-05 | In Git |

问题描述

在远端创建一个项目时，选中使用README初始化项目。 两外在本地创建一个项目，创建文件并提交修改。 此时将本地的代码提交到远程仓库，执行以下代码后提示错误

结果提示：

git refusing to merge unrelated histories

<!-- more -->

解决方法

问题原因是本地的 master 分支上有修改，远程仓库的 master 分支有修改，但它们没有共同的父节点。所以 git 不知道如何处理。

解决方法是：

更新 git pull origin master -u --allow-unrelated-histories

之前版本

  1. 通过 git fetch origin master 将远程代码保存到本地的 origin/master 分支

  2. 使用 git rebase origin/master 将远程代码和本地代码进行合并，代码会提示以下信息

    First, rewinding head to replay your work on top of it... Applying: init Using index info to reconstruct a base tree... Falling back to patching base and 3-way merge...

可以看到内部操作是，回头重新执行版本的初始化工作，先进行初始化，使用索引信息重构 base tree，回到修补基地进行3路合并。

意思是 我重新创建一个空项目，然后将远程和本地的两个版本合并到主线上。
